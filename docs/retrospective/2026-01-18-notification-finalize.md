# 2026-01-18: 알림 시스템 마무리 회고

## 문제 개요

**증상(사용자 관점)**
- 승인/공지/알림장 이벤트가 실제 알림 목록에 충분히 반영되지 않는다.
- 드롭다운의 기본 필터/삭제 정책이 명확하지 않다.

**영향**
- 사용자가 알림 신뢰도가 떨어지고, 주요 액션 확인이 늦어진다.

---

## 문제 원인

### 1) 이벤트별 알림 생성 연결 부족
- 승인/거절은 일부 연결되어 있었으나 공지/알림장은 생성 로직이 누락되었다.

### 2) 삭제/읽음 정책 전달 부족
- 읽음은 보관, 삭제는 숨김이라는 정책이 UI/문서에 명확히 반영되지 않았다.

---

## 해결/개선 조치

### 1) 알림 생성 포인트 보강
- 공지사항 생성 시 유치원 구성원(원장/교사/학부모)에게 알림 전송.
- 알림장 작성 시 대상 학부모에게 알림 전송(원생/반/전체별 분기).

### 2) 드롭다운 UX 개선
- 기본 미읽음 필터를 명시하고, 배지/목록 갱신을 즉시 반영.
- 삭제는 soft delete로 숨김 처리, 읽음은 상태만 변경.

---

## 향후 방지책

- 알림 생성 로직을 공통 이벤트로 추상화해 누락을 방지한다.
- 통합 알림 테스트 시나리오를 문서화한다.
