# 유치원 관리 시스템 구축 로직

## 1. 사용자 역할 및 초기 진입
- **원장(Director)**: 유치원 생성 및 관리 총괄
- **교사(Teacher)**: 유치원 소속 신청 후 승인 대기
- **학부모(Parent)**: 원아 등록 후 유치원 신청 및 승인 대기

## 2. 상세 워크플로우

### [원장]
1. 회원가입/로그인
2. 소속된 유치원 유무 확인
   - 없으면: `유치원 생성 페이지`로 강제 이동
   - 생성 정보: 유치원명, 주소, 연락처, 사업자번호 등 (AI가 필요 항목 추가 정의)
3. 생성 완료 시 시스템 등록 완료

### [교사]
1. 회원가입/로그인
2. 유치원 검색/선택 (DB에 등록된 목록 기반, 오름차순 정렬)
3. 등록 신청 -> 원장에게 알림 발송
4. 승인 전까지 `등록 대기 페이지` 유지
5. 승인 후 `유치원 관리 페이지` 진입 (원장보다 적은 권한)

### [학부모]
1. 회원가입/로그인
2. `원아 정보 입력 페이지` 이동 (이름, 생년월일, 성별, 특이사항 등)
3. 유치원 검색/선택 후 등록 신청
4. 원장 또는 교사의 승인/거절 대기
5. 승인 후 유치원/원아 정보 확인 및 소통(댓글 등) 가능
6. 유치원에서 알림, 공지 등이 올라오면 알림

### 알람 기능
1. 위에서 언급했던것처럼 원장, 선생님, 학부모 모드 알람 기능 필요. 
2. 알림은 읽지 않으면 상단 알림 로고나 버튼에 빨간점으로 아래 표시
3. 읽거나 삭제했으면 빨간점 사라짐
